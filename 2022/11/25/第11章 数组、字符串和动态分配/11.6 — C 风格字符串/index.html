11.6 — C 风格字符串

在第4.17课– std::string 简介中，我们将字符串定义为连续字符的集合，例如“Hello, world!”。字符串是我们在 C++ 中处理文本的主要方式，而 std::string 使在 C++ 中处理字符串变得容易。

现代 C++ 支持两种不同类型的字符串：std::string（作为标准库的一部分）和 C 风格的字符串（从 C 语言继承而来）。事实证明，std::string 是使用 C 风格的字符串实现的。在本课中，我们将仔细研究 C 风格的字符串。

C-style字符串

C-style的字符串只是一个使用空终止符的字符数组。空终止符( null terminator**)是用于指示字符串结束的特殊字符（’\0’，ascii 码 0）。更一般地说，C 风格的字符串称为**空终止字符串(null-terminated string)**。

需要注意的重要一点是 C-style的字符串遵循与数组相同的所有规则。这意味着您可以在创建字符串时对其进行初始化，但之后您不能使用赋值运算符为其赋值！

因为并非所有编译器都支持 strcpy_s()，strlcpy() 是一种流行的替代方法——尽管它是非标准的，因此没有包含在很多编译器中。它也有自己的一系列问题。简而言之，如果您需要复制 C 风格的字符串，这里没有普遍推荐的解决方案。

不要使用 C 风格的字符串

了解 C 风格的字符串很重要，因为它们在很多代码中使用。但是，既然我们已经解释了它们的工作原理，我们将建议您尽可能完全避免使用它们！除非您有特定的、令人信服的理由使用 C 风格的字符串，否则请改用 std::string（在 标头中定义）。std::string 更简单、更安全、更灵活。在极少数情况下，您确实需要使用固定的缓冲区大小和 C 风格的字符串（例如，对于内存有限的设备），我们建议使用为此目的设计的经过良好测试的第 3 方字符串库，或者std::string_view，它是而是在下一课中介绍。
