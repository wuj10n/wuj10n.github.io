11.4 — 使用选择排序对数组进行排序

排序案例

对数组进行排序是按特定顺序排列数组中所有元素的过程。在许多不同的情况下，对数组进行排序很有用。例如，您的电子邮件程序通常按收到时间的顺序显示电子邮件，因为较新的电子邮件通常被认为更相关。当您转到联系人列表时，姓名通常按字母顺序排列，因为这样更容易找到您要查找的姓名。这两种演示都涉及在演示前对数据进行排序。

对数组进行排序可以提高搜索数组的效率，不仅对人类如此，对计算机也是如此。例如，考虑我们想知道一个名字是否出现在一个名字列表中的情况。为了查看某个名称是否在列表中，我们必须检查数组中的每个元素以查看该名称是否出现。对于包含许多元素的数组，搜索所有元素的代价可能很高。

但是，现在假设我们的姓名数组是按字母顺序排序的。在这种情况下，我们只需要搜索到遇到按字母顺序大于我们要查找的名称的名称为止。到那时，如果我们还没有找到这个名字，我们就知道它不存在于数组的其余部分，因为我们没有在数组中查看的所有名字都保证按字母顺序排列更大！

事实证明，还有更好的算法来搜索排序的数组。使用一个简单的算法，我们可以仅使用 20 次比较来搜索包含 1,000,000 个元素的排序数组！当然，缺点是对数组进行排序相对昂贵，而且通常不值得对数组进行排序以加快搜索速度，除非您要多次搜索它。

在某些情况下，对数组进行排序可以使搜索变得不必要。考虑另一个我们想要找到最佳考试成绩的例子。如果数组未排序，我们必须遍历数组中的每个元素以找到最大的测试分数。如果列表是排序的，最好的测试分数将在第一个或最后一个位置（取决于我们是按升序还是降序排序），所以我们根本不需要搜索！

排序的工作原理

排序通常是通过重复比较成对的数组元素来执行的，如果它们满足某些预定义的条件，则交换它们。这些元素的比较顺序因使用的排序算法而异。标准取决于列表的排序方式（例如升序或降序）。

选择排序

有很多方法可以对数组进行排序。选择排序可能是最容易理解的排序，这使它成为教学的一个很好的候选者，尽管它是速度较慢的排序之一。

选择排序执行以下步骤将数组从小到大排序：

  1. 从数组索引 0 开始，搜索整个数组以找到最小值
  2. 将数组中找到的最小值与索引 0 处的值交换
  3. 从下一个索引开始重复步骤 1 和 2

换句话说，我们要找到数组中最小的元素，并将其交换到第一个位置。然后我们要找到下一个最小的元素，并将它交换到第二个位置。将重复此过程，直到我们用完所有元素。

这是该算法处理 5 个元素的示例。让我们从一个示例数组开始：

{ 30, 50, 20, 10, 40 }

首先，我们找到最小的元素，从索引 0 开始：

{ 30, 50, 20, 10, 40 }

然后我们将其与索引 0 处的元素交换：

{ 10, 50, 20, 30, 40 }

现在第一个元素已经排序，我们可以忽略它。现在，我们找到最小的元素，从索引 1 开始：

{ 10, 50, 20, 30, 40 }

并将其与索引 1 中的元素交换：

{ 10, 20, 50, 30, 40 }

现在我们可以忽略前两个元素。找到从索引 2 开始的最小元素：

{ 10, 20, 50, 30, 40 }

并将其与索引 2 中的元素交换：

{ 10, 20, 30, 50, 40 }

找到从索引 3 开始的最小元素：

{ 10, 20, 30, 50, 40 }

并将其与索引 3 中的元素交换：

{ 10, 20, 30, 40, 50 }

最后，找到从索引 4 开始的最小元素：

{ 10, 20, 30, 40, 50 }

并将它与索引 4 中的元素交换（它什么都不做）：

{ 10, 20, 30, 40, 50 }

完毕！

{ 10, 20, 30, 40, 50 }

请注意，最后一次比较将始终与自身进行比较（这是多余的），因此我们实际上可以在数组末尾之前停止 1 个元素。
