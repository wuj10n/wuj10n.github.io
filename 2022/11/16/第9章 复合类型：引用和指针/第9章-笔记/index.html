复合类型：引用和指针

9.1复合数据类型介绍

什么是复合类型？

在计算机科学中，复合类型是一种数据类型，它可以原始类型和其它的复合类型所构成。

C++ 支持以下复合类型？

  • Functions 函数

  • Arrays 数组

  • Pointer types 指针类型:

    • Pointer to object 指向对象的指针
    • Pointer to function 函数指针

  • Pointer to member types 指向成员类型的指针:

    • Pointer to data member 指向数据成员的指针
    • Pointer to member function 指向成员函数的指针

  • Reference types 引用类型:

    • L-value references L 值引用
    • R-value references R 值引用

  • Enumerated types 枚举类型:

    • Unscoped enumerations无范围枚举
    • Scoped enumerations 范围枚举

  • Class types 类类型:

    • Structs 结构体
    • Classes 类
    • Unions 联合

9.2值类别（左值和右值）

表达式2个属性？

类型

计算表达式生成结果的值、对象或函数的类型 int / int => int

表达式的类型必须在编译时可以确定（否则类型检查和类型推导将不起作用）

但是，表达式的值可以在编译时（如果表达式是 constexpr）或运行时确定（如果表达式不是 constexpr）。

值类别

在 C++11 之前，只有两种可能的值类别：lvalue和rvalue.

在 C++11 中，添加了三个额外的值类别（glvalue、prvalue和xvalue）以支持名为move semantics（移动语义）.

左值和右值是什么？

左值(lvalue) 是B语言/C语言/C++语言等类C语言中的一类表达式。“左”(left)的原意是指可以放在赋值符号“=”的左边，但其实也表示能作为&和++等操作符的操作数（B语言中已经如此）。而且，现代C/C++中的含义已经不局限于此。lvalue的l被重新解释为location。这也对应于ISO C11/ISO C++11的内存位置(memory location)。

确切的说 C++ 中左值和右值的概念是从 C 语言继承过来的值得一提的是，左值的英文简写为“lvalue”，右值的英文简写为“rvalue”。很多人认为它们分别是”left value”、”right value” 的缩写，其实不然。

lvalue 是“loactor value”的缩写，可意为存储在内存中、有明确存储地址（可寻址）的数据，rvalue 译为 “read value”，指的是那些可以提供数据值的数据（不一定可以寻址，例如存储于寄存器中的数据）。

左值会隐式转换为右值，所以左值可以在需要右值的地方使用。

左值表达式是那些解析为变量或其他可识别对象的表达式，这些对象在表达式末尾后仍然存在。
右值表达式是那些计算为文字或在表达式末尾丢弃的函数和运算符的返回值的表达式。

9.3左值引用

现代 C++ 包含两种类型的引用：左值引用和右值引用。

左值引用必须绑定到可修改的左值。

左值引用不能绑定到不可修改的左值或右值（否则您将能够通过引用更改这些值，这将违反它们的常量性）。出于这个原因，左值引用有时被称为非常量左值引用（有时简称为非常量引用）。

一旦初始化，C++ 中的引用就不能重新设置，这意味着它不能更改为引用另一个对象。

References 和 referents 有独立的生命周期

  • 引用可以在其引用的对象之前销毁。
  • 被引用的对象可以在引用之前销毁。 会导致悬挂引用

9.4对 const 的左值引用

左值引用如何只能绑定到可修改的左值，对于一个 const 变量，我们想要创建一个引用怎么办？

对 const值的左值引用可以绑定到不可修改的左值。

对 const 的左值引用也可以绑定到可修改的左值。在这种情况下，被引用的对象在通过引用访问时被视为 const（即使底层对象是非const量）。

对 const 的左值引用也可以绑定到右值，将创建一个临时对象并使用右值进行初始化，并且对 const 的引用将绑定到该临时对象。

为了避免在这种情况下出现悬空引用，C++ 有一个特殊的规则：当一个 const 左值引用绑定到一个临时对象时，临时对象的生命周期会延长以匹配引用的生命周期。

9.5通过左值引用传递

使用按引用传递时，我们将函数参数声明为引用类型（或 const 引用类型）而不是普通类型。调用函数时，每个引用参数都绑定到适当的参数。因为引用充当参数的别名，所以不会创建参数的副本

通过引用传递给非const形参，形参只能接受可修改的左值参数。

与对非常量的引用（它只能绑定到可修改的左值）不同，对 const 的引用可以绑定到可修改的左值、不可修改的左值和右值。因此，如果我们将引用参数设为 const，那么它将能够绑定到任何类型的参数。

通过 const 引用传递提供与通过引用传递相同的主要好处（避免制作参数的副本），同时还保证函数不能更改被引用的值。

为什么允许用右值初始化对const左值引用？

without that capability, there would be no way to pass literals (or other rvalues) to functions that used pass by reference!
